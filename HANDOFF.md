## 5. Bring-up checklist (colleague, day one)

### Frontend

```
cd Desktop/pets24x7_new

# Just to view local. Double-clicks start_local.bat OR:
python -m http.server 8000
# Visit http://localhost:8000/
```

To regenerate the 35k SEO pages after CSV updates (only if `../Pets24x7_DATA/` is available):

```
python build_data.py    # rebuilds pets-data.js + data/*.json
python build_pages.py   # regenerates in/, us/, sitemap.xml
```

### Backend

```
cd Desktop/pets24x7_api
cp .env.example .env       # fill DATABASE_URL, JWT_SECRET, WA_*, PHONEPE_*
npm install                # 217 packages
npm run prisma:gen
npm run prisma:migrate     # creates tables on Supabase
npm run seed:admin         # creates first admin from .env
npm run seed:plans         # creates 6 membership plans
npm run dev                # http://localhost:4000
```

Health check: `curl http://localhost:4000/health` → `{ ok: true }`.

Admin panel: http://localhost:4000/admin/login → log in with `SEED_ADMIN_EMAIL` + `SEED_ADMIN_PASSWORD` from `.env`.

### End-to-end smoke test

1. Front: http://localhost:8000/parent-login/ → enter your phone → OTP → dashboard.
2. Front: http://localhost:8000/vendor-login/ → enter a phone present in `data/*.json` → match → OTP → dashboard (PENDING status).
3. Admin: http://localhost:4000/admin/vendors?status=PENDING → click **Approve**.
4. Front: refresh vendor dashboard → banner flips to ACTIVE.
5. Front: http://localhost:8000/membership/ → pick a plan → redirected to PhonePe sandbox → pay with test card → land on `/membership/return/` → success → membership shown on parent dashboard.
6. Admin: `/admin/memberships` and `/admin/payments` rows visible.

## 6. Deploy targets (production)

**Frontend → Hostinger**
- See `pets24x7_new/HOSTINGER-DEPLOY.md`
- Pack: `pack_for_hostinger.bat` → upload `pets24x7-deploy.zip` to `public_html/` → extract.
- `.htaccess` handles clean URLs + 301 legacy redirects + force HTTPS + force apex.

**Backend → Railway**
- See `pets24x7_api/README.md` §4
- `railway up`, set env vars, add custom domain `api.pets24x7.com`.
- Add CNAME `api → <project>.up.railway.app` at the DNS provider.

**Database → Supabase**
- Free tier covers 5,000 vendors comfortably.
- Use pooler URI (port 6543) in `DATABASE_URL`, not direct (5432).

## 9. Two ways to hand off

### Option A — Git (recommended for ongoing work)

```
# in each folder
cd Desktop/pets24x7_new && git init && git add . && git commit -m "initial: phase 1+2 frontend"
cd Desktop/pets24x7_api  && git init && git add . && git commit -m "initial: phase 1+2 backend"

# push to two private GitHub/GitLab repos
gh repo create pets24x7-frontend --private --source=. --push
gh repo create pets24x7-backend  --private --source=. --push

# add colleague as collaborator on both repos in GitHub UI
```

Pros: version-controlled, diffable, every commit auditable, deploys hook into Railway/Netlify directly.

### Option B — Single zip

```
cd Desktop
pack_handoff.bat        # produces pets24x7-handoff.zip (~5-10 MB)
```

Send the zip via Google Drive / Dropbox / WeTransfer. Include this `HANDOFF.md` and `pets24x7_api/.env.example` (NOT the real `.env`).

Pros: zero infra needed. Cons: no version history, must re-zip on every update.
